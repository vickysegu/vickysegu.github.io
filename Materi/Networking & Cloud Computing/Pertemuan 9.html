Blok 01 · Pengantar

1. Pengantar: Memahami Ulang Komputasi Awan

Membangun server fisik secara tradisional memerlukan belanja modal awal (CapEx) yang sangat besar untuk pengadaan perangkat keras, penyediaan fasilitas pendingin, serta pemeliharaan oleh teknisi ahli. Teknologi komputasi awan (cloud computing) merevolusi paradigma ini dengan menawarkan infrastruktur virtual berbasis sewa yang fleksibel, sehingga mengubah beban pengeluaran menjadi biaya operasional (OpEx) yang efisien dan dapat disesuaikan kebutuhan.

💡 Konsep Abstraksi

Penyediaan sumber daya komputasi (server, database, jaringan) secara virtual melalui internet berdasarkan permintaan (on-demand), tanpa perlu memiliki infrastruktur fisik sendiri.

🎯 Transformasi Biaya (OpEx)

Mengeliminasi biaya besar pembelian server di awal (CapEx). Bisnis hanya membayar apa yang digunakan secara bulanan (OpEx) dan dapat melakukan penskalaan kapasitas dalam hitungan menit.

🚀 Metode Deployment

Admin masuk ke konsol penyedia layanan awan (seperti AWS, GCP, atau DigitalOcean), mengonfigurasi spesifikasi server virtual (VPS), dan langsung menjalankannya saat itu juga.

Setelah memahami pergeseran model bisnis dari fisik ke awan, bagaimana cara kita membedakan berbagai model layanan awan yang disewakan di pasar? Mari kita tinjau analogi sederhananya.

Blok 01 · Pengantar

2. Jembatan Teori: Analogi Properti

Memahami tiga klasifikasi utama layanan awan—Software as a Service (SaaS), Platform as a Service (PaaS), dan Infrastructure as a Service (IaaS)—melalui analogi berjenjang industri properti dan restoran, guna mempermudah pemetaan batas tanggung jawab pemeliharaan, tingkat kontrol konfigurasi, serta fleksibilitas pengelolaan antara penyedia layanan cloud dengan pengguna akhir.

🟩 SaaS - Aplikasi Siap Pakai

Deskripsi: Layanan aplikasi siap pakai di internet.

Kelebihan: Pengguna tinggal menggunakan fitur tanpa pusing menulis kode atau konfigurasi server.

Contoh: Buka browser, login, langsung pakai seperti Google Workspace, Canva, Gmail.

🟪 PaaS - Platform Siap Deploy

Deskripsi: Penyewaan platform untuk menjalankan kode aplikasi.

Kelebihan: Pengembang fokus 100% pada penulisan logika aplikasi tanpa mengurus pemeliharaan OS.

Contoh: Upload kode aplikasi, platform menjalankannya otomatis seperti Google App Engine, Heroku.

🟥 IaaS - Infrastruktur Mentah

Deskripsi: Penyewaan server virtual mentah, jaringan, dan firewall.

Kelebihan: Memberikan kebebasan mutlak bagi admin untuk mengatur sistem operasi dan keamanan.

Contoh: Sewa VPS kosong, instal OS Linux secara manual, konfigurasi dari nol di AWS EC2 atau DigitalOcean Droplet.

Blok 02 · Memilih Layanan

3. Mengapa Kita Memilih IaaS?

Meskipun model SaaS dan PaaS menawarkan kepraktisan instan tanpa beban pemeliharaan sistem dasar, model Infrastructure as a Service (IaaS) tetap menjadi fondasi utama bagi arsitek infrastruktur. IaaS memberikan kebebasan penuh bagi administrator untuk mengontrol alokasi sumber daya, menentukan sistem operasi, memodifikasi kernel, mengonfigurasi jaringan, serta menerapkan kebijakan keamanan secara mandiri.

🖥️ Lapisan Virtualisasi

Infrastruktur awan mentah berupa mesin virtual (Virtual Machine) yang diabstraksikan dari server fisik oleh lapisan software bernama Hypervisor .

🎯 Kebebasan Otoritas

Memberikan kendali absolut . Anda bebas menentukan OS, memodifikasi aturan dinding api (firewall), menginstal dependensi khusus, dan tidak memiliki batasan dari penyedia cloud.

⚙️ Operasional Administratif

Admin mengelola VPS menggunakan akses administrator tertinggi ( root / superuser ) melalui antarmuka perintah teks (CLI) dari jarak jauh.

Dengan kebebasan kendali penuh atas server virtual (VPS) di tangan Anda, parameter teknis apa saja yang wajib ditentukan saat pertama kali menyewa? Mari kita bedah detail arsitekturnya.

Blok 03 · Arsitektur

4. Parameter Arsitektur: Spesifikasi & Lokasi Data Center

Sebelum melakukan peluncuran (deployment) server, penentuan spesifikasi perangkat keras virtual (CPU, RAM, dan Storage) serta pemilihan lokasi geografis pusat data fisik (data center) merupakan langkah perencanaan taktis. Keputusan arsitektur ini berdampak langsung pada kecepatan transmisi data (latensi jaringan (latency)) yang dirasakan oleh pengguna akhir serta efisiensi pengeluaran anggaran operasional (OpEx) bisnis.

📊 Alokasi Hardware

Penentuan kapasitas virtual hardware (vCPU, RAM, SSD) yang sesuai dengan kebutuhan komputasi web Anda untuk menghindari pemborosan.

🌐 Dampak Jarak Fisik

Spesifikasi berlebih (overprovisioning) membuang anggaran (OpEx tinggi). Lokasi server yang jauh menimbulkan Penundaan Propagasi (latensi) karena jarak tempuh kabel optik.

⚙️ Penerapan Praktis

Pilih spesifikasi paling hemat (Basic) untuk aplikasi ringan. Pilih data center terdekat (misal Jakarta) untuk melayani pengguna lokal Indonesia agar latensi minimal.

Begitu spesifikasi hardware dan lokasi server disepakati, keputusan krusial berikutnya adalah memilih sistem operasi. Mengapa platform minimalis Linux menguasai ekosistem ini?

Blok 03 · Arsitektur

5. Dominasi Linux pada Lingkungan Awan

Hampir seluruh infrastruktur awan skala global di dunia berjalan di atas distribusi sistem operasi Linux seperti Ubuntu, Debian, atau Red Hat. Sifatnya yang open-source meniadakan biaya lisensi sistem operasi, sementara efisiensi penggunaan memori, stabilitas jangka panjang yang teruji, serta fleksibilitas kustomisasi baris perintah menjadikannya standar industri di pusat data modern.

🐧 Sistem Tanpa GUI

Linux Server adalah sistem operasi sumber terbuka (open-source) minimalis tanpa antarmuka grafis (GUI) yang dikendalikan murni lewat baris perintah.

💰 Efisiensi & Nol Lisensi

Sistem GUI Windows memakan RAM & CPU besar hanya untuk render desktop. Linux membebaskan 100% hardware untuk beban kerja aplikasi. Linux juga gratis tanpa lisensi jutaan rupiah.

🛠️ Pengelolaan CLI

Instal OS Linux Server ringan pada VPS, lalu lakukan instalasi modul perangkat lunak menggunakan paket manager bawaan (seperti apt di Ubuntu/Debian).

Kini, setelah seluruh landasan teori arsitektur awan terpenuhi, saatnya Anda menguji langsung pemahaman ini dengan merancang dan mendeploy mesin virtual pertama Anda.

Blok 04 · Alat Kerja

7. Alat Kerja Utama: Terminal (CLI)

Berbeda dari komputer pribadi yang menggunakan antarmuka grafis (GUI) yang memakan banyak memori, server produksi di awan dikendalikan sepenuhnya tanpa tampilan visual grafis. Baris perintah berbasis teks murni (Command Line Interface atau CLI) digunakan sebagai media kendali jarak jauh yang sangat cepat, hemat bandwidth, dan memfasilitasi otomatisasi perintah administrasi secara efisien.

💻 Antarmuka Teks (CLI)

Command Line Interface (CLI) / Terminal adalah program pembungkus (shell) yang mengeksekusi instruksi teks murni langsung ke inti sistem operasi.

⚡ Kecepatan & Efisiensi

Sangat hemat bandwidth internet, sangat cepat dieksekusi, mudah diotomatisasi lewat script, dan tidak membebani performa memori server.

🛠️ Akses Aplikasi

Gunakan aplikasi PowerShell/CMD di Windows, atau Terminal di macOS/Linux untuk terhubung ke server awan (Lihat terminal simulator yang telah aktif di kanan layar).

Blok 04 · Alat Kerja

8. Terowongan Kriptografi: Protokol SSH

Mengelola server awan melintasi jaringan internet publik memerlukan protokol komunikasi yang sangat aman untuk mencegah intersepsi data. Protokol Secure Shell (SSH) membentuk saluran kriptografi terenkripsi yang melindungi lalu lintas perintah administratif dari ancaman penyadapan (eavesdropping) maupun serangan man-in-the-middle oleh pihak yang tidak bertanggung jawab.

🔒 Secure Shell (SSH)

Secure Shell (SSH) is a cryptographic network protocol for operating network services securely over an unsecured network on standard Port 22 .

🛡️ Proteksi Eksploitasi

Protokol lama seperti Telnet mengirim password dalam bentuk teks biasa (plaintext) sehingga rawan disadap. SSH mengamankan seluruh sesi komunikasi dengan enkripsi kriptografi.

⚙️ Instruksi Koneksi

Ketik perintah: ssh [user]@[IP_Publik] .
Contoh: ssh ubuntu@103.45.1.2 .

Blok 05 · Operasional

9. Hukum Pertama Admin: Patching Keamanan

Tindakan preventif pertama yang wajib dijalankan oleh administrator sistem sesaat setelah berhasil masuk ke server baru adalah memperbarui indeks paket repositori lokal dan melakukan pembaruan keamanan (patching). Langkah ini sangat krusial untuk menambal kerentanan bawaan pada template OS Image dan melindungi server baru dari pemindaian botnet otomatis yang agresif.

🔄 Pembaruan Paket (Patching)

Proses memperbarui seluruh pustaka paket dan sistem operasi server ke versi rilis keamanan paling mutakhir untuk menambal celah eksploitasi.

🛡️ Mitigasi Kerentanan

OS Image awal dari penyedia cloud berasal dari template lama. Menunda update membuat server rentan diserang oleh botnet pemindai IP otomatis dalam hitungan jam.

⚙️ Eksekusi Perintah

Ketik perintah pembaruan repositori dan paket:
sudo apt update && sudo apt upgrade -y

Sistem operasi Anda kini telah aman dan mutakhir. Namun, server kosong belum bernilai bagi bisnis sebelum kita memasang layanan yang dapat diakses oleh pengunjung.

Blok 05 · Operasional

10. Membangun Nilai Bisnis (Deployment Layanan)

Sebuah server virtual yang telah diamankan tidak akan memberikan nilai fungsi komersial sebelum dikonfigurasi untuk melayani kebutuhan bisnis. Melalui instalasi dan pemetaan port layanan latar belakang (service atau daemon) seperti web server (Apache/NGINX) atau database engine (MariaDB), server tersebut baru dapat memproses dan merespons permintaan transaksi digital dari pelanggan.

⚙️ Layanan Latar Belakang (Daemon)

Daemon: Program komputer yang berjalan di latar belakang (background) sistem, menunggu dan merespons request jaringan pada port tertentu.

🌐 Web Server (Port 80)

Merespons request HTTP/HTTPS peramban klien pada port 80/443 untuk menyajikan dokumen halaman web ke pengunjung.

🗄️ Database Server (Port 3306)

Berfungsi sebagai penyimpanan data relasional terstruktur untuk database transaksi, stok kopi, dan akun pengguna.

Setelah memahami cara kerja port dan program latar belakang (daemon), mari kita praktikkan langsung proses instalasi dan deployment layanan web di terminal Anda.

Blok 07 · Praktikum Langsung

11. Final Misi: Eksekusi Pemasangan Layanan!

Lakukan proses instalasi dan konfigurasi layanan web server atau database server secara riil pada mesin virtual Kopi Senja Anda menggunakan simulator terminal. Evaluator kecerdasan buatan (XAI) akan menganalisis kecocokan pilihan perangkat lunak Anda dengan skenario operasional dan anggaran bisnis ritel harian.

🟩 Web Server NGINX

Definisi: Web Server asinkron berperforma tinggi.

Kelebihan: Arsitektur event-driven membuat NGINX sangat hemat RAM dan CPU dalam melayani request secara bersamaan.

Perintah: sudo apt install nginx -y

🟥 Web Server Apache

Definisi: Web Server tradisional berbasis thread.

Kelebihan: Menggunakan arsitektur process-driven (satu thread per request) yang sangat modular dan handal.

Perintah: sudo apt install apache2 -y

🟦 Database MariaDB

Definisi: Sistem database relasional SQL open source.

Kelebihan: Alternatif MySQL yang lebih cepat dan memiliki performa kueri relasional tinggi untuk data stok dan akun.

Perintah: sudo apt install mariadb-server -y

Blok 06 · Troubleshooting

12. Troubleshooting Jaringan & Simulasi Eror

Menganalisis kegagalan koneksi jaringan dan kesalahan konfigurasi SSH secara sistematis melalui pembacaan kode galat (error logs) di terminal. Simulasikan skenario kendala autentikasi atau kesalahan alamat host menggunakan modul interaktif di bawah untuk mempelajari cara mendiagnosis dan memperbaiki gangguan koneksi server (troubleshooting) secara mandiri.

💡 Pelacakan Masalah

Menganalisis pesan kesalahan (error output) untuk menemukan letak kegagalan komunikasi jaringan.

Aktifkan simulasi di sebelah kanan untuk mengamati respons terminal secara interaktif.

📌 Tips: Simulasi otomatis mendeploy server bayangan IP 103.45.1.2 jika belum dideploy.

❌ Connection Timed Out

Masalah: Firewall memblokir gerbang Port 22 sehingga paket TCP diabaikan (*dropped*).

Solusi: Centang/Buka Port 22 (SSH) di firewall dashboard cloud.

❌ Permission Denied

Masalah: Koneksi menembus port 22, namun modul autentikasi menolak akses karena username salah (seperti memaksa login root ).

Solusi: Ubah username login ke default OS Linux (seperti ubuntu ).
